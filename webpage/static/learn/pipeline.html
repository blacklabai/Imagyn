How to use imagyn and run processes in a pipeline

Process pipelining

Imagyn uses Spotify's Luigi to manage all of the tasks that are required to run through the process of assembling a new dataset. Luigi helps us configure and address issues typically associated with long-running batch processes. Issues that come up are caching, monitoring, and automating. Luigi does what GNU Make does, but it's all implemented in python and provides support for a few of the common automation needs - reporting, repeatability, ease of documentation, and onwards.

A simple way to get going with Luigi is to just run the example tasks from our repository. These tasks will try to download some of images with your keyword in them, some images related but not the same as your keyword, and some completely unrelated images. After that's complete, a random selection of transformations will be applied to the images we've retrieved. You can then go ahead an begin training your models to recognize the words you've asked for.

To get the visualizer demo running, perform the following:
Assuming you already have luigi installed, and your PYTHONPATH environment variable includes the directory where imagyn is installed) $> luigid This will get the visualizer server running, the luigid command will be available after luigi is installed.
Next just request to run a task:

$> python ./project/luigi/pipeline.py RunAll
.> --keyword toucan --imgCount 20 --exact 70 --similar 20 --unrelated 10
