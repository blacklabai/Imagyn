Who is Echosec?

Echosec is a tech startup company based in Victoria. Their location-based search platform provides public safety, security, journalism, and intelligence professionals actionable knowledge based on social media and other information.

Learn more

What is UVic SENG 499?

Also known as Design Project 2, SENG 499 is a course required to be taken by Software Engineering undergraduate students. The course requires students to pursue an independent project under the supervision of a faculty member, to prepare a written report and present a seminar describing the work. Projects will normally focus on large software systems, and collaboration with an industrial sponsor is encouraged.

Learn more

What are we building?

We've chosen to build Echosec a framework to help them automate the process of assembling and augmenting training and testing data for the neural applications they employ as part of their product. Echosec uses machine learning to detect items in photos they encounter while processing a client's search request. Currently these machine learning applications are trained on data which is collected and annotated by hand. Our project, Imagyn is a framework which automatically collects annotated images for training a neural network, trains the network, informs you of the overall progress, and hands back an image recognizer ready to identify what you're requested.

Learn more

Imagyn
/iˈmajən/

Imagyn has several parts, here's a little breakdown about each one of them.

Image Retrieval

We use ImageNet as our source of annotated images. ImageNet is based off of a library called WordNet: a "large lexical database of English. Nouns, verbs, adjectives and adverbs are grouped into sets of cognitive synonyms (synsets), each expressing a distinct concept". ImageNet simply maps images to each synset in WordNet. We then retrieve the images from ImageNet's API.

Learn more

Image Synthesis

Using various software packages, we modify the images that are collected by applying various transformations. Transformations include adding noise, rotating, skewing, and carving to name a few. Transformations are applied to help the neural network that will train on these images recognize occurrences of them when they're not represented as nearly as in other training data.

Learn more

Model Training

Imagyn was built to collect images to train an image recognizer, so we've done that as well.

Learn more

Pipelining and UI

Imagyn uses Spotify's Luigi to manage all of the tasks that are required to run through the process of assembling a new dataset. Luigi helps us configure and address issues typically associated with long-running batch processes.

Learn more
